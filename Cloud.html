📖 CORE DEFINITION

Cloud Computing

Cloud Computing (CC) is the on-demand delivery of computing resources (storage, processing, apps) over the internet, allowing users to scale and pay only for what they use — without owning physical infrastructure.
On-DemandScalablePay-as-you-goElastic

⏳ HISTORY

Evolution Timeline

1950s: Mainframes + Time-Sharing (users take turns).
1972: IBM VM/370 — first VM-supporting OS.
1999: Salesforce — first SaaS.
2006: AWS EC2 — true cloud era begins.
2008: Google App Engine (PaaS) + Azure launch.

🚌 DEPLOYMENT MODELS

Cloud Models

Public 🚌 Bus — shared, low cost, high scale.
Private 🚗 Car — dedicated, full control, secure.
Hybrid 🚖 Taxi — public + private combined.
Community 🚐 Carpool — shared between similar orgs.

🛠️ SERVICE MODELS

IaaS / PaaS / SaaS

IaaS — user manages OS, middleware, apps, data.
PaaS — user manages only apps and data.
SaaS — vendor manages everything.
On-Prem — you own/manage all 9 layers.
AWS EC2HerokuGmail

🖥️ HYPERVISOR & VIRTUALIZATION

Hypervisors

Hypervisor: manages multiple VMs on one physical host.
Type 1 (bare-metal): runs directly on hardware. VMware ESXi, Hyper-V, Xen.
Type 2 (hosted): runs on top of an OS. VirtualBox, VMware Workstation.
VM: isolated, full OS, strong security. Slow start (minutes).

📦 CONTAINERS

Docker & Kubernetes

Container: lightweight, shares host kernel, starts in seconds.
Docker: build/run containers. Dockerfile defines image. Docker Hub = registry.
Kubernetes (K8s): orchestrates containers — scaling, rolling updates, self-healing. Pod = group of containers.
Faster than VMsLightweightK8s by Google

🔒 SECURITY

Cloud Security

Shared Responsibility: provider secures physical infra; customer secures data, OS, apps.
Zero Trust: "Never trust, always verify." Every request authenticated.
MFA: Multi-Factor Authentication — extra security layer.
Encryption: protects data confidentiality in transit and at rest.
DoS: Denial-of-Service — aims to disrupt, NOT steal data.

⚙️ MANAGEMENT TOOLS

Config & Monitoring

Ansible: YAML playbooks, agentless, SSH-based config management.
Chef: Ruby DSL, uses agents (chef-client), recipes & cookbooks.
Puppet: declarative language, agent-based, master-agent model.
Nagios: monitoring/alerting tool.
Zabbix: all-in-one monitoring — metrics, alerts, dashboards.

🌐 ARCHITECTURE

Edge, Fog, CDN, SOA

Edge Computing: process data near the source (low latency).
Fog Computing: intermediate layer (routers/gateways) between edge and cloud.
CDN: Content Delivery Network — caches and accelerates content delivery.
SOA: Service-Oriented Architecture — loosely coupled services via APIs (REST).
FaaS: Function as a Service = serverless (AWS Lambda). No idle charges.

🏗️ DATA CENTER HARDWARE

Servers & Infrastructure

Tower: standalone, ideal for small offices.
Rack: 1U = 1.75 inches. Mounted in cabinets.
Blade: requires chassis/enclosure, high density.
PDU: Power Distribution Unit — distributes electricity to racks.
UPS: Uninterruptible Power Supply — backup power.
Cold/Hot Aisle: data center cooling design for efficiency.

💾 STORAGE TYPES

Block / File / Object

Block Storage: raw volumes, like hard drives. Fast. (AWS EBS, Cinder).
File Storage: shared file system (NFS, EFS).
Object Storage: unstructured data with metadata. (AWS S3, Swift). Best for large scale.
Snapshot: point-in-time copy — revert VM/disk state.

🔧 OPENSTACK

Private Cloud Platform

Nova: compute (VMs).
Neutron: networking.
Cinder: block storage.
Swift: object storage.
Glance: VM image management.
Keystone: identity/authentication.
OpenStack is open-source software for private clouds — NOT a public cloud provider.
