📦 CH1 — Systèmes Embarqués

DÉFINITION FONDAMENTALE

Système Embarqué

Système embarqué [Kadionik 2004] : Combinaison matériel + logiciel dédiée à une ou plusieurs fonctions spécifiques avec des contraintes sévères : consommation, température, taille, performances.
Dispositif mobile : Appareil informatique portable possédant un écran et une interface E/S. Inclut : laptop, tablet PC, PDA, smartphone, MP3.
DédiéContraintAutonome

CONTRAINTES CLÉS

Les 5 Contraintes

1. Faible coût : Solution optimale prix/performance. ~6$/processeur (2004).
2. Faible consommation : Batterie, autonomie max, pas de gaspillage.
3. Faible encombrement : Composants compacts sur faible surface.
4. Temps réel : Réponse rapide aux événements, critique en contrôle.
5. Environnement : Température, humidité, vibrations, radiations RF.

CHIFFRES CLÉS

Marché des Processeurs

✅ 98% des processeurs vendus → embarqué (95% selon sources)
✅ 2–5% → PC
✅ Prix moyen embarqué : 6$ vs 300$ PC (2004)

En 2004 : 14 milliards de processeurs pour l'embarqué, 260 millions pour PC.

CARTES PROGRAMMABLES

Arduino / Raspberry / Micro:bit

Arduino (2005) : Licence libre, langage C-like, câble USB, simple.
Raspberry Pi (2012) : Python, sans-fil, OS intégré, plus puissant.
Micro:bit (2015) : Éducatif, capteurs intégrés, matrice LEDs, Python.

HISTORIQUE

Repères Chronologiques

1967 : 1er système embarqué — guidage Apollo.
1971 : 1er microprocesseur Intel commercialisé.
1982 : 1er objet connecté — distributeur Coca-Cola.
1984 : Airbus A320 — commandes électroniques.
2007 : iPhone Apple — emblème du smartphone.
2014 : Google Car — radar, caméras, GPS.
📱 CH2 — OS Mobile & Architecture Android

DÉFINITION

OS Mobile

OS Mobile : Logiciel permettant à un terminal mobile (smartphone, PDA, tablette, smartwatch) de fonctionner. Gère : connectivité sans fil (WiFi, Bluetooth, GPS), appels, téléchargement d'apps, personnalisation.
Parts de marché 2023 : Android 44,6% · Windows 28,41% · iOS 17,29%
Propriétaire: iOSOpen Source: Android

INTRODUCTION

Android — Origine

Android : Plateforme mobile = OS + librairies middleware (C/C++) + applications. Basé sur kernel Linux 2.6. Framework en Java.
2005 : Google acquiert Android Inc.
2007 : Lancement de l'OHA (Open Handset Alliance).
2008 : Premier standard Android annoncé.
OHA : Alliance d'opérateurs (Vodafone), fabricants (HTC, LG), semi-conducteurs (Intel, ARM), éditeurs (Google).

ARCHITECTURE

Les 5 Couches Android

1. Applications — Email, SMS, Maps, Navigateur (Java/Kotlin)
2. App Framework — Activity Mgr, Content Providers, Notification Mgr, Location Mgr, Package Mgr, Window Mgr, Telephony Mgr, Resource Mgr, View System (Java)
3. Librairies — Bionic libc, SQLite, WebKit, OpenGL ES, SSL, SGL, SurfaceManager, Media Framework, FreeType (C/C++)
4. Android Runtime — ART (ex-Dalvik), Core Libraries (.dex)
5. Noyau Linux — Drivers, Binder IPC, Ashmem, Wakelock (C)

COMPOSANT 2

Intent

Intent : Objet de messagerie pour demander une action. Attributs : Nom, Action, Data (URI/MIME), Catégorie, Extras (clé/valeur), Flags.
Explicite : cible nommée directement → navigation interne à une app.
Implicite : action décrite → système trouve le composant → dialog "Ouvrir avec" si plusieurs matchs.
IntentFilter : dans le manifest, spécifie quels intents un composant peut gérer (action, category, data).

COMPOSANTS 3, 4, 5

Service / Broadcast / ContentProvider

Service : Tâche arrière-plan sans UI. Types : Foreground (notification visible), Background (API 26+ restreint), Bound (client-serveur Binder). Ex: lecteur musique.
BroadcastReceiver : Reçoit Intents diffusés. onReceive() doit être COURT (thread UI). NE PEUT PAS lancer Activity, Dialog, ou Service long.
ContentProvider : Partage données entre apps via URIs (content://authority/path). Méthodes : query(), insert(), update(), delete(). Cursor = itérateur sur résultats.
⚙️ CH4 — Processus, Threads & CFS Scheduling

THREADING ANDROID

Main Thread & Worker Threads

Main Thread (UI Thread) : Lancé au démarrage. Exécute : callbacks du cycle de vie, BroadcastReceiver.onReceive(), méthodes d'événements UI.
⚠️ RÈGLE DES 5 SECONDES : Ne jamais bloquer le main thread > 5s → ANR (Application Not Responding)
Solutions pour modifier l'UI depuis un worker :
• Activity.runOnUiThread(Runnable)
• View.post(Runnable)
• View.postDelayed(Runnable, long)
• Handler (niveau supérieur, file de messages)
• AsyncTask (déprécié API 30)

CFS SCHEDULING

Completely Fair Scheduler

CFS : Algorithme depuis Linux 2.6.23 pour processus SCHED_NORMAL. Utilise un arbre bicolore (red-black tree) trié par vruntime. Élit toujours le nœud le plus à GAUCHE (vruntime minimal).
W(nice) ≈ 1024 / 1.25^nice
W₀ (nice=0) = 1024
Quantum_i = (W_i / ΣW) × TL
ΔVruntime = Δt × (W₀ / W_i)
→ Processus élu : vruntime le plus PETIT
Nice : -20 (priorité max) → +19 (priorité min). Défaut = 0.

🧮 EXERCICE CFS COMPLET — 3 threads (nice: -5, 0, +5), TL=3ms, dt=1ms, ΣW=3121+1024+335=4480

Quantum T1 (nice=-5, W=3121) = (3121/4480) × 3 ≈ 2.09 ms
Quantum T2 (nice=0,  W=1024) = (1024/4480) × 3 ≈ 0.69 ms
Quantum T3 (nice=+5, W=335)  = (335/4480)  × 3 ≈ 0.22 ms

ΔVruntime T1 (Δt=1ms) = 1 × (1024/3121) ≈ 0.328 ms  → vruntime augmente lentement (haute priorité)
ΔVruntime T2 (Δt=1ms) = 1 × (1024/1024) = 1.000 ms  → référence
ΔVruntime T3 (Δt=1ms) = 1 × (1024/335)  ≈ 3.057 ms  → vruntime augmente vite (basse priorité)

→ Ordre d'élection : T1 (vruntime le plus petit) est élu en premier.
→ À chaque tick : mise à jour de vruntime, l'arbre bicolore est réorganisé.
→ Au réveil : vruntime ← max(vruntime_processus, min_vruntime) pour éviter la famine.
💾 CH5 — Gestion de la Mémoire

STRUCTURE RAM

Pages Mémoire (4KB)

Cached privé :
• Original (non modifié) → libéré directement par kswapd
• Modifié → compressé en zRAM par kswapd
Cached partagé :
• Original → libéré directement
• Modifié → écrit en storage (kswapd, msync(), munmap())
Anonyme (MAP_ANONYMOUS) :
• Modifié → compressé en zRAM
zRAM : Partition de RAM compressée pour le swap. Pages anonymes/modifiées compressées ici.

MÉTRIQUES MÉMOIRE

RSS / PSS / USS

RSS (Resident Set Size) : Privé + TOUT le partagé. SURESTIME.
PSS (Proportional Set Size) : Privé + partagé/n. MEILLEURE MÉTRIQUE ⭐
USS (Unique Set Size) : Privé seulement. SOUS-ESTIME.
PSS(app) = pages_privées + Σ(pages_partagées / nb_utilisateurs)

RÉCUPÉRATION MÉMOIRE

kswapd → onTrimMemory → LMK

Garbage Collector (CMS) : Young generation → Older generation → Permanent generation. Multi-thread, pauses courtes.
kswapd : Démon kernel. Actif si RAM libre < seuil min. Arrêt si RAM libre > seuil max (high watermark).
LMK : Dernier recours. Utilise oom_adj_score. Tue : Background (LRU d'abord) → Service → Visible → Foreground (dernier).
⚠️ kswapd ne s'arrête PAS quand la RAM est pleine. Il s'arrête quand la RAM libre DÉPASSE le seuil MAXIMUM.

🧮 EXERCICE PSS COMPLET (TD43)

Apps : Within (3 pages privées), GMaps (2 pages privées), Viber (1 page privée)
Location Service : 4 pages PARTAGÉES

CAS 1 — Seulement GMaps utilise le Location Service :
  PSS(Within)  = 3 + 0       = 3 pages
  PSS(GMaps)   = 2 + 4/1     = 6 pages
  PSS(Viber)   = 1 + 0       = 1 page
  Total pages  = 3+2+4+1 = 10 pages × 4KB = 40KB

CAS 2 — GMaps ET Viber partagent le Location Service :
  PSS(Within)  = 3 + 0       = 3 pages
  PSS(GMaps)   = 2 + 4/2     = 4 pages
  PSS(Viber)   = 1 + 4/2     = 3 pages
  Total pages  = 3+2+4+1 = 10 pages × 4KB = 40KB (inchangé)

RAM = 50KB, Utilisée = 40KB → RAM libre = 10KB
Seuil minimal = 12KB → RAM libre (10KB) < seuil (12KB)
→ kswapd activé + si insuffisant → LMK tue processus background (Within en premier car LRU)

RÈGLE PSS DE WITHIN : Within n'utilise JAMAIS Location Service → PSS(Within) = 3 dans TOUS les cas !

DOZE MODE & STANDBY

Économie d'Énergie

Doze Mode : Déclenché si stationnaire + écran OFF + débranché. Restreint : réseau, CPU intensif, jobs, alarmes, syncs, wakelocks. Fenêtres maintenance à intervalles décroissants.
Quitte Doze si : mouvement détecté, écran ON, ou appareil en charge.
App Standby Buckets (API 28+) :
Active (aucune restriction) → Working Set (restriction moyenne) → Frequent (forte) → Rare (stricte) → Never (sévère)
Quittent le standby dès que l'appareil est en charge.
