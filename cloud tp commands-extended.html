🖥️ VMware & Virtual Networking NAT/Bridged/Host-Only
# Verify virtualization is enabled (Windows: Task Manager > Performance > CPU)
Check if "Virtualization: Enabled" — required for VMware/VirtualBox.
# On Linux host: egrep -c '(vmx|svm)' /proc/cpuinfo
Returns >0 if CPU virtualization extensions (VT-x/AMD-V) active.
# Inside Linux Lite VM: ip a or ifconfig
Show all network interfaces, IP addresses, MAC.
ip route show default
Display default gateway (ex: 192.168.x.1).
cat /etc/resolv.conf
View DNS server addresses configured for VM.
ping -c 4 8.8.8.8
Test internet connectivity via Google DNS (ICMP).
ping -c 4 google.com
Check DNS resolution + external reachability.
ping <host_ip>
From VM → host machine (depends on network mode).
# On host: ping <VM_IP>
Test connectivity host → VM in bridged or host-only.
# SSH config inside Linux Lite: sudo apt update && sudo apt install openssh-server -y
Install SSH server to allow remote connections.
systemctl status ssh
Check if SSH daemon is active and running.
ssh localhost
Test SSH locally — if works, server is ready.
sudo hostnamectl set-hostname LinuxLite-02
Rename machine hostname permanently.
ssh user@<target_vm_ip>
Establish SSH session from one VM to another.
# Export VM as OVA: In VMware → File → Export OVF Template
Package entire VM for backup or cloning.

🐳 Docker Fundamentals images · containers · run
docker -v
Show Docker client version (installed check).
docker version
Detailed client + server engine version info.
docker run hello-world
Download test image & run container → "Hello from Docker" confirmation.
docker ps
List RUNNING containers (ID, image, status, ports).
docker ps -a
Show ALL containers (including stopped/exited).
docker images
List all downloaded images (local cache).
docker image rm <image_id_or_name>
Remove image (e.g., docker image rm fedora).
docker container run redis
Pull Redis image & run in foreground (attached).
docker container run -d redis
Run Redis DETACHED mode (background).
docker container run -d redis:alpine
Run specific lightweight Alpine-based Redis image.
docker inspect <container>
Show low-level JSON details (IP, mounts, config, state).
docker logs <container_id>
Fetch container's STDOUT / STDERR logs.
docker stats
Live resource usage (CPU, MEM, NET I/O) of running containers.
docker info
System-wide info: containers, images, storage driver, etc.
docker rm <container_name>
Remove stopped container (use -f to force remove running).
docker container prune
Remove all stopped containers to free space.
🐍 Containerizing Python Apps Dockerfile · matplotlib · volumes
mkdir hello-docker && cd hello-docker
Create project directory for Python app.
# app.py: print("Hello, Docker!")
Simple python script.
# Dockerfile: FROM python:3.8-slim COPY app.py /app.py CMD ["python","/app.py"]
Define base image, copy script, default command.
docker build -t hello-docker .
Build image from Dockerfile (tag: hello-docker).
docker run hello-docker
Run container → prints "Hello, Docker!".
docker tag hello-docker yourusername/hello-docker
Tag local image for Docker Hub registry.
docker login
Authenticate to Docker Hub (interactive).
docker push yourusername/hello-docker
Upload image to Docker Hub repository.
# Matplotlib app: mkdir matplotlib-docker && cd matplotlib-docker
Project for generating graphs without display.
# Dockerfile: FROM python:3.11-slim, RUN pip install matplotlib, COPY plot_graph.py . RUN mkdir /output
Install matplotlib, create output folder.
docker build -t matplotlib-app .
Build matplotlib image.
docker run -v ${PWD}/output:/output matplotlib-app
Mount volume to persist generated graph.png locally.
# Windows CMD: docker run -v %cd%/output:/output matplotlib-app
Bind mount current directory in Windows CMD.
ls output/
Verify graph.png has been created.
docker rmi matplotlib-app
Remove image by name / ID.
🌐 Multi-Container Web App WordPress · MySQL · Adminer
docker network create demo
Create custom bridge network for inter-container communication.
docker pull mysql:8
Download MySQL 8 image from Docker Hub.
docker run --network demo --name db -e MYSQL_ROOT_PASSWORD=password -e MYSQL_DATABASE=wordpress -d mysql:8
Run MySQL container with root pass and auto-create wordpress db.
docker logs db
Check MySQL startup logs (ready to accept connections).
docker pull adminer:4
Pull Adminer (lightweight DB management tool).
docker run --network demo -d -p 8080:8080 adminer:4
Run Adminer exposed on host port 8080.
docker pull wordpress:5
Official WordPress image version 5.
docker run --network demo -d -p 8000:80 --name wordpress-container wordpress:5
WordPress container accessible on http://localhost:8000.
# Access Adminer: http://localhost:8080 (Server: db, user: root, pass: password)
Visual database management interface.
docker stop db wordpress-container
Gracefully stop containers.
docker rm db wordpress-container
Remove stopped containers.
docker network rm demo
Delete custom network.

💾 Volumes · Backup · Kubernetes PV persistent storage
docker volume ls
List all Docker volumes (named & anonymous).
docker volume create my_data
Create a named volume (managed by Docker).
docker volume inspect my_data
Show metadata: mountpoint, driver, options.
docker run -d --name my_container -v my_data:/app/data ubuntu:latest sleep infinity
Mount volume into container at /app/data (persistent).
docker exec my_container sh -c "echo 'Hello from volume' > /app/data/test.txt"
Write data inside volume via container.
docker exec my_container cat /app/data/test.txt
Read file content from volume.
docker run -d --name another_container -v my_data:/app/shared_data ubuntu:latest sleep infinity
Second container sharing same volume (different mount path).
docker stop my_container another_container && docker rm my_container another_container
Stop+remove containers before backup.
# Backup volume to tar archive (Windows CMD: %cd%)
docker run --rm -v my_data:/source:ro -v %cd%:/backup ubuntu tar cvf /backup/my_data_backup.tar -C /source .
docker volume rm my_data
Delete original volume (data still in backup).
docker volume create my_restored_data
Create fresh volume for restoration.
docker run --rm -v my_restored_data:/dest -v %cd%:/backup ubuntu bash -c "tar xvf /backup/my_data_backup.tar -C /dest"
Extract backup into new volume.
docker run --rm -v my_restored_data:/app/data ubuntu cat /app/data/test.txt
Verify restored file content.
# Kubernetes PV / PVC (optional extension)
kubectl apply -f pv.yaml (PersistentVolume manifest)
kubectl get pv
List persistent volumes in cluster.
kubectl apply -f pvc.yaml
Create PersistentVolumeClaim to request storage.
kubectl exec pod-demo -- cat /data/test.txt
Check mounted volume inside pod.
🧹 Cleanup & Utility Commands docker system prune
docker system prune -a
Remove all unused containers, networks, images (caution).
docker stop $(docker ps -aq)
Stop all running containers.
docker rm $(docker ps -aq)
Remove all containers (stopped or running with -f).
docker image prune -f
Delete dangling images.
docker volume prune -f
Remove unused local volumes (free disk).
docker network ls
Show all Docker networks (bridge, host, none, custom).
docker network inspect demo
Details about network: connected containers, subnet.
docker exec -it <container> /bin/bash
Interactive shell inside running container.
docker logs --tail 50 <container>
Show last 50 log lines.
docker run --rm -it python:3.8-slim bash
Ephemeral container with interactive bash (removed after exit).
git clone <repo-url>
Clone GitHub repository locally (used with Codespaces).
chmod 777 output/
Fix permission denied for volume mounts (Linux/Mac).
