Cloud Computing & Virtualization Commands Cheat Sheet

This file collects all commands from TP1–TP7 (VMware, Codespaces, Docker, Docker Compose, Kubernetes) with short explanations.

TP1 - VMware Network Modes

Ping VM -> Host
Tests connectivity from VM to host machine.
Ping Host -> VM
Tests connectivity from host to VM.
Ping Gateway
Checks if VM can reach its gateway.
Ping 8.8.8.8
Tests external connectivity (Google DNS).
Ping google.com
Tests DNS resolution and external connectivity.

TP2 - VM Deployment & SSH

sudo apt update && sudo apt install openssh-server
Installs OpenSSH server inside VM.
systemctl status ssh
Checks if SSH service is running.
ssh localhost
Tests SSH connection to self.
ping <VM_IP>
Ping between host and VM interfaces.
ssh user@LinuxLite-01
SSH into VM from host.
ssh user@LinuxLite-02
SSH from one VM into another.

TP3 - GitHub Codespaces & Apache

sudo apt update
Updates package list.
sudo apt install apache2 -y
Installs Apache web server.
sudo service apache2 start
Starts Apache service.
sudo service apache2 status
Checks Apache status.
sudo nano /etc/apache2/apache2.conf
Edits Apache config file.
ServerName localhost
Fixes ServerName warning.
cd /var/www/html
Navigates to web root.
sudo nano index.html
Edits default web page.
sudo service apache2 restart
Restarts Apache after changes.

TP4 - Docker Basics

mkdir hello-docker && cd hello-docker
Creates project directory.
docker build -t hello-docker .
Builds Docker image.
docker run hello-docker
Runs container from image.
docker images
Lists Docker images.
docker ps -a
Lists all containers (running + stopped).
docker login
Logs into Docker Hub.
docker tag hello-docker user/hello-docker
Tags image for Docker Hub.
docker push user/hello-docker
Pushes image to Docker Hub.

TP6 - Docker Compose

docker-compose up --build
Builds and starts services.
docker-compose ps
Shows running services.
docker-compose down
Stops and removes containers, networks.

TP7 - Docker Volumes & Kubernetes

docker volume ls
Lists volumes.
docker volume create my_data
Creates named volume.
docker volume inspect my_data
Shows volume details.
docker run -d --name my_container -v my_data:/app/data ubuntu sleep infinity
Runs container with mounted volume.
docker exec my_container sh -c "echo 'Hello from Docker volume' > /app/data/test.txt"
Writes file into volume.
docker exec my_container cat /app/data/test.txt
Reads file from volume.
docker run -d --name another_container -v my_data:/app/shared_data ubuntu sleep infinity
Shares volume with another container.
docker stop my_container another_container
Stops containers.
docker rm my_container another_container
Removes containers.
docker run --rm -v my_data:/source:ro -v $(pwd):/backup ubuntu tar cvf /backup/my_data_backup.tar -C /source .
Backs up volume.
docker volume rm my_data
Deletes volume.
docker volume create my_restored_data
Creates new volume.
docker run --rm -v my_restored_data:/dest -v $(pwd):/backup ubuntu bash -c "tar xvf /backup/my_data_backup.tar -C /dest"
Restores backup into new volume.
kubectl apply -f pv.yaml
Creates PersistentVolume in Kubernetes.
kubectl apply -f pvc.yaml
Creates PersistentVolumeClaim.
kubectl apply -f pod.yaml
Deploys pod using PVC.
kubectl exec pod-demo -- cat /data/test.txt
Verifies data inside pod.
